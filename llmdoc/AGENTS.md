# llmdoc 知识库 (AI Agent 专用)

## OVERVIEW
ACadSharp 结构化文档系统，为 AI 代理提供多维度的项目上下文和精准的检索映射。

## STRUCTURE
- **overview/**: 核心业务逻辑与对象模型的高级视图。
- **architecture/**: 系统架构细节，作为 AI 的“检索地图”揭示底层执行流与设计模式。
- **guides/**: 任务导向的实战指南，解决“如何实现特定 CAD 操作”的流程问题。
- **reference/**: 事实性参考资料，包含编码规范、版本兼容性及特定系统规格。
- **agent/**: 深度扫描报告，包含由代理生成的详细代码映射、分析与测试用例。

## CONVENTIONS
- **文档驱动开发 (DDD)**: 代理在修改代码前应至少阅读 3 份相关文档以建立上下文。
- **结构化引用**: 所有文档间通过相对路径互联，确保“文档丛林”的连通性。
- **LLM 友好格式**: 优先使用 Markdown 表格描述参数，使用 Mermaid 图表描述逻辑流。
- **同步维护原则**: 代码变更后，必须同步使用 `recorder` 代理更新受影响的文档章节。
- **分类严谨性**: 新文档必须归入上述五大目录之一，禁止随意放置在根目录。

## ANTI-PATTERNS
- **冗余父级内容**: 禁止在此目录下重复项目根目录 `AGENTS.md` 或 `README.md` 的内容。
- **扁平化倾向**: 避免将所有文档堆放在单一文件夹，必须遵循五层分类架构。
- **断开的索引**: 增加新文档或重命名后，未同步更新 `index.md` 中的导航链接。
- **描述不清的报告**: `agent/` 中的报告应包含具体的文件路径、代码片段和明确的结论。
- **忽略版本**: 记录 IO 或实体行为时，未指明对应的 CAD 版本（AC1009-AC1032）。
